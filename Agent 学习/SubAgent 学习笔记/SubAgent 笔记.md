# SubAgent 设计准则

1. SubAgent 只做一件事情
2. SubAgent 只做工具调用
3. 对话由主 Agent 处理

> [!info]
>  主模型做对话是免费的能力——它本来就在和用户聊天，不需要额外调度。subagent
  做的是需要工具（Read/Grep）的脏活。各取所长。
  

```

┌─────────────────────────────────────────────────────┐

│  何时用 Sub-agent？                                  │

│                                                     │

│  ✅ 工具密集型：Read / Write / Glob / Grep           │

│  ✅ 边界清晰：输入输出可定义为结构化数据             │

│  ✅ 可并行：任务间无数据依赖                         │

│  ✅ 纯执行：不需要判断/决策/权衡                     │

│                                                     │

│  何时留在主模型？                                    │

│                                                     │

│  ✅ 需要和用户对话                                   │

│  ✅ 需要综合多源信息做判断                           │

│  ✅ 需要创造性方案组合                               │

│  ✅ 需要做选择/决策（选方案A或B）                    │

│  ✅ 需要保持上下文连贯                               │

└─────────────────────────────────────────────────────┘

```

> [!quote]
> SubAgent 只在"**这件事主模型做要等很久**"或"**这件事需要和主模型当前工作隔离**"时使用。
