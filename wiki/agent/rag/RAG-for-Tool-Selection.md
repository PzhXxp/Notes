# RAG 在工具选择中的应用

## 核心思想

传统的 RAG 用于从知识库中检索相关文本来增强 LLM 的回答。**RAG-MCP 将这一思想迁移到工具选择领域**：不是检索知识，而是检索工具描述。

- 传统 RAG：知识检索（文档段落）→ 增强生成
- RAG-MCP：**工具检索（API Schema）→ 增强工具调用**

## 为什么需要 RAG 做工具选择？

当 LLM 可用的外部工具数量增长到成百上千时（如 MCP 生态已有 4,400+ 服务器），将所有工具描述放入 Prompt 会导致：

1. **Prompt Bloat**：上下文窗口被工具描述填满
2. **决策干扰**：相似功能的工具让 LLM 难以区分
3. **幻觉增加**：模型可能幻觉出不存在的 API

RAG 通过语义检索只选出最相关的工具，从根本上解决这些问题。

## 工具检索的实现

- **向量索引**：将所有 MCP 工具的元数据（名称、描述、参数 schema）编码为向量存入外部索引
- **语义匹配**：用户查询同样编码为向量，通过余弦相似度匹配最相关的工具
- **动态注入**：仅将 Top-1（或 Top-k）工具描述注入 Prompt

## 关键优势

- **无需重训**：新工具只需添加到索引中，LLM 本身不需要重新训练
- **跨领域泛化**：检索器不局限于特定领域，任何有语义描述的工具都可检索
- **与 MCP 天然契合**：MCP 协议的标准化 schema 使得工具描述的索引和检索非常自然
